System Architecture: Triple-Model Ensemble

Three complementary deep learning models working in concert for robust plant health assessment

🎯 Model 1: MobileNetV2 (Disease Classifier)

Purpose: Multi-class disease identification with specific diagnosis

Performance: 93.33% accuracy, 100% precision, 96.49% F1

Architecture: Lightweight CNN (1.0× depth, 224×224). Source: ozair23/mobilenet_v2_1.0_224-finetuned-plantdisease (HuggingFace)

Training Data: PlantVillage dataset (38 disease classes)

Advantage: Identifies specific disease types (e.g., "Early Blight" vs "Late Blight") with high precision.

⚖️ Model 2: ResNet50 (Binary Health Classifier)

Purpose: Robust Healthy/Diseased binary classification via transfer learning

Architecture: Deep Residual Network (50 layers) + Custom Binary Head

Approach: Pretrained on ImageNet → Feature extraction → Binary classification head (FC 2048→512→2)

Advantage: Transfer learning captures general leaf health patterns. Provides independent validation of Model 1's disease status classification.

🤝 Model 3: Ensemble Consensus

Purpose: Cross-model validation and agreement analysis

Methodology: Compares binary predictions from Models 1 & 2

Output: AGREE ✅ (high confidence) or DISAGREE ⚠️ (requires review)

Advantage: Production-grade ensemble learning. Agreement between independent models significantly increases prediction reliability. Disagreement flags uncertain cases.

🏭

Why Three Models? (Production AI Best Practice)

Modern production AI systems use ensemble methods to improve robustness. Our approach mirrors industry standards: Model 1 specializes in disease classification (high precision, specific diagnosis). Model 2 validates overall health status through transfer learning (independent architecture, different learning paradigm). Model 3 provides agreement metrics that flag high-confidence predictions vs uncertain cases requiring expert review. This redundancy improves reliability and provides explainability.

Why do published papers show 99%?

High-accuracy papers (Mohanty 2016, Ferentinos 2018) test on the same controlled PlantVillage images used for training — clean, studio-lit, uniform backgrounds. Real-world generalisation is consistently lower. Our ~70% extended result on mixed-source real images is in line with what independent replication studies report for MobileNetV2 on out-of-distribution leaf photos.

Comprehensive Evaluation Methodology

Triple-model system evaluation — fully transparent and reproducible

🎯 Model 1: MobileNetV2

Source: ozair23/mobilenet_v2_1.0_224-finetuned-plantdisease (HuggingFace Transformers). No additional fine-tuning — pretrained weights used as-is.

Input: 224×224 RGB images

Output: 38-class disease labels with confidence scores

⚖️ Model 2: ResNet50

Source: PyTorch torchvision.models (pretrained on ImageNet). Custom binary classification head appended.

Architecture: 50-layer ResNet → Identity function (feature extraction) → FC(2048)→ReLU→Dropout(0.3)→FC(2)

Output: Binary classification (Healthy / Diseased) with probability scores

🤝 Model 3: Ensemble

Logic: Compares health status from Model 1 (extracted from disease label) and Model 2 (direct binary output).

Agreement Criterion: If both models predict same health status → AGREE ✅

Output: Consensus badge, model disagreement flags, confidence metrics

🖼️ Test Dataset

30 images (10 per class) from PlantVillage public dataset. Not used in model training splits. Balanced class distribution.

Classes: Pepper Bell Healthy, Potato Early Blight, Tomato Healthy

⚙️ Preprocessing Pipeline

1. Resize to 224×224
2. Convert to tensor
3. ImageNet normalisation (mean [0.485,0.456,0.406], std [0.229,0.224,0.225])
4. Batch inference

📐 Metrics Computation

scikit-learn: accuracy_score, precision_score, recall_score, f1_score (weighted average). Label mapping applied to align folder names with model outputs.

Ensemble Metrics: Agreement percentage, confidence boost analysis

🔁 Reproducibility

Evaluation script evaluate_model.py in repository. Run python evaluate_model.py --test_dir test_images/ to reproduce all Model 1 metrics. Ensemble validation performed live in web interface.

⚠️ System Limitations

Model 1: Small test set (30 images), 3 of 38 classes. Late Blight variants excluded. Real-world generalisation limited.

Model 2: Requires good image quality. Transfer learning may not capture all disease-specific features.

Ensemble: Limited by individual model accuracy. Disagreement requires expert human review.
